#### 1.5 Auto-Export on Session(s) End â­ NEW

Automatically prompt or create ticket artifacts when chat session(s) end. Supports combining multiple related sessions into a single ticket.

**Settings:**

```json
{
  "ticketManager.autoExport": {
    "enabled": true,
    "trigger": "session_end",      // "session_end" | "idle_timeout" | "manual_only"
    "idleTimeoutMinutes": 30,      // For idle_timeout trigger
    "behavior": "prompt",          // "prompt" | "auto_draft" | "auto_create"
    "minimumMessages": 5,          // Don't trigger for short chats
    "defaultExportType": "request", // "request" | "tasks" | "plan" | "all"
    "autoDetectType": true,        // Smart detection of artifact type
    "excludePatterns": [           // Don't trigger for certain conversations
      "quick question",
      "how do I",
      "what is"
    ]
  }
}
```

**Behavior Options:**

| Behavior | Description |
|----------|-------------|
| `prompt` | Show dialog: "Save this conversation as a ticket?" |
| `auto_draft` | Silently create draft ticket, notify user |
| `auto_create` | Create ticket immediately with extracted data |
| `manual_only` | Never auto-trigger, user must run `/export` |

**Trigger Options:**

| Trigger | When It Fires |
|---------|---------------|
| `session_end` | User closes chat panel or starts new conversation |
| `idle_timeout` | No messages for X minutes |
| `context_switch` | User opens different file/project |
| `manual_only` | Only via `/export` command |

**Smart Detection:**

The extension analyzes conversation to determine:
1. **Should export?** - Was this a substantive work conversation?
2. **What type?** - Bug fix, feature, investigation, or general Q&A?
3. **Existing ticket?** - Append to related ticket or create new?
